1. Tổng quan mô hình dữ liệu

Hệ thống được tổ chức theo các nhóm chính:

  • Danh mục: product_groups, products, warehouses
  • Tồn kho hiện tại: inventories
  • Chứng từ nhập/xuất/chuyển/kiểm kê: stock_receipts, stock_issues, stock_transfers, stocktakes và các bảng chi tiết tương ứng
  • Nhật ký biến động kho: inventory_transactions
  • Người dùng & phân quyền: users, roles, permissions và bảng trung gian

2. Sơ đồ quan hệ (dạng văn bản dễ dán Word)

Quan hệ Loại Mô tả
product_groups.id → products.product_group_id 1 - N Một nhóm sản phẩm có nhiều sản phẩm
warehouses.id → inventories.warehouse_id 1 - N Một kho có nhiều dòng tồn kho
products.id → inventories.product_id 1 - N Một sản phẩm xuất hiện ở nhiều kho
warehouses.id → stock_receipts.warehouse_id 1 - N Chứng từ nhập gắn với một kho
stock_receipts.id → stock_receipt_items.stock_receipt_id 1 - N Một phiếu nhập có nhiều dòng chi tiết
products.id → stock_receipt_items.product_id 1 - N Một sản phẩm có thể nằm trong nhiều phiếu nhập
warehouses.id → stock_issues.warehouse_id 1 - N Chứng từ xuất gắn với một kho
stock_issues.id → stock_issue_items.stock_issue_id 1 - N Một phiếu xuất có nhiều dòng chi tiết
products.id → stock_issue_items.product_id 1 - N Một sản phẩm có thể nằm trong nhiều phiếu xuất
warehouses.id → stock_transfers.from_warehouse_id 1 - N Kho nguồn trong chứng từ chuyển kho
warehouses.id → stock_transfers.to_warehouse_id 1 - N Kho đích trong chứng từ chuyển kho
stock_transfers.id → stock_transfer_items.stock_transfer_id 1 - N Một phiếu chuyển có nhiều dòng chi tiết
products.id → stock_transfer_items.product_id 1 - N Sản phẩm trong dòng chuyển kho
warehouses.id → stocktakes.warehouse_id 1 - N Phiếu kiểm kê gắn với một kho
stocktakes.id → stocktake_items.stocktake_id 1 - N Một phiếu kiểm kê có nhiều dòng kiểm kê
products.id → stocktake_items.product_id 1 - N Sản phẩm trong dòng kiểm kê
warehouses.id → inventory_transactions.warehouse_id 1 - N Nhật ký giao dịch theo kho
products.id → inventory_transactions.product_id 1 - N Nhật ký giao dịch theo sản phẩm
users.id → (stock_receipts, stock_issues, stock_transfers, stocktakes, inventory_transactions).created_by 1 - N Người tạo chứng từ/giao dịch

3. Từ điển dữ liệu chi tiết

3.1. Bảng users

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Định danh người dùng
name string NOT NULL Họ tên
email string UNIQUE, NOT NULL Email đăng nhập
email_verified_at timestamp NULL Thời điểm xác thực email
password string NOT NULL Mật khẩu đã hash
is_active boolean DEFAULT true Trạng thái hoạt động
remember_token string NULL Token nhớ đăng nhập
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.2. Bảng product_groups

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã nhóm sản phẩm
code string UNIQUE Mã nhóm
name string NOT NULL Tên nhóm
description text NULL Mô tả
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.3. Bảng products

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã sản phẩm
product_group_id bigint FK → product_groups.id, NULL Nhóm sản phẩm
code string UNIQUE Mã sản phẩm
name string INDEX Tên sản phẩm
unit string NOT NULL Đơn vị tính
min_stock_alert decimal(18,3) DEFAULT 0 Ngưỡng cảnh báo tối thiểu
shelf_life_days unsigned int NULL Hạn sử dụng (ngày)
description text NULL Mô tả sản phẩm
is_active boolean DEFAULT true Trạng thái hoạt động
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.4. Bảng warehouses

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã kho
code string UNIQUE Mã kho
name string NOT NULL Tên kho
address string NOT NULL Địa chỉ kho
description text NULL Mô tả
is_active boolean DEFAULT true Trạng thái
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.5. Bảng inventories

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã dòng tồn kho
warehouse_id bigint FK → warehouses.id Kho
product_id bigint FK → products.id Sản phẩm
quantity decimal(18,3) DEFAULT 0 Số lượng tồn hiện tại
last_movement_at timestamp NULL Lần biến động gần nhất
created_at, updated_at timestamp Thời gian tạo/cập nhật
Unique key: (warehouse_id, product_id)

3.6. Bảng stock_receipts (phiếu nhập)

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã phiếu nhập
code string UNIQUE Mã chứng từ nhập
warehouse_id bigint FK → warehouses.id Kho nhập
receipt_date date INDEX Ngày nhập
note text NULL Ghi chú
created_by bigint FK → users.id, NULL Người tạo
total_amount decimal(18,2) DEFAULT 0 Tổng tiền
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.7. Bảng stock_receipt_items

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã dòng chi tiết
stock_receipt_id bigint FK → stock_receipts.id Thuộc phiếu nhập nào
product_id bigint FK → products.id Sản phẩm
quantity decimal(18,3) NOT NULL Số lượng
unit_cost decimal(18,2) DEFAULT 0 Đơn giá nhập
line_total decimal(18,2) DEFAULT 0 Thành tiền dòng
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.8. Bảng stock_issues (phiếu xuất)

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã phiếu xuất
code string UNIQUE Mã chứng từ xuất
warehouse_id bigint FK → warehouses.id Kho xuất
issue_date date INDEX Ngày xuất
note text NULL Ghi chú
created_by bigint FK → users.id, NULL Người tạo
total_amount decimal(18,2) DEFAULT 0 Tổng tiền
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.9. Bảng stock_issue_items

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã dòng chi tiết
stock_issue_id bigint FK → stock_issues.id Thuộc phiếu xuất nào
product_id bigint FK → products.id Sản phẩm
quantity decimal(18,3) NOT NULL Số lượng
unit_price decimal(18,2) DEFAULT 0 Đơn giá xuất
line_total decimal(18,2) DEFAULT 0 Thành tiền dòng
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.10. Bảng stock_transfers (phiếu chuyển kho)

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã phiếu chuyển
code string UNIQUE Mã chứng từ chuyển
from_warehouse_id bigint FK → warehouses.id Kho nguồn
to_warehouse_id bigint FK → warehouses.id Kho đích
transfer_date date INDEX Ngày chuyển
note text NULL Ghi chú
created_by bigint FK → users.id, NULL Người tạo
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.11. Bảng stock_transfer_items

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã dòng chi tiết
stock_transfer_id bigint FK → stock_transfers.id Thuộc phiếu chuyển nào
product_id bigint FK → products.id Sản phẩm
quantity decimal(18,3) NOT NULL Số lượng chuyển
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.12. Bảng stocktakes (phiếu kiểm kê)

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã phiếu kiểm kê
code string UNIQUE Mã chứng từ kiểm kê
warehouse_id bigint FK → warehouses.id Kho kiểm kê
checked_at date INDEX Ngày kiểm kê
note text NULL Ghi chú
apply_adjustment boolean DEFAULT false Áp dụng điều chỉnh tồn
created_by bigint FK → users.id, NULL Người tạo
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.13. Bảng stocktake_items

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã dòng kiểm kê
stocktake_id bigint FK → stocktakes.id Thuộc phiếu kiểm kê nào
product_id bigint FK → products.id Sản phẩm kiểm kê
system_qty decimal(18,3) NOT NULL Số lượng hệ thống
actual_qty decimal(18,3) NOT NULL Số lượng thực tế
variance_qty decimal(18,3) NOT NULL Chênh lệch (thực tế - hệ thống)
created_at, updated_at timestamp Thời gian tạo/cập nhật

3.14. Bảng inventory_transactions

Cột Kiểu Ràng buộc Ý nghĩa
id bigint PK Mã giao dịch kho
warehouse_id bigint FK → warehouses.id Kho phát sinh
product_id bigint FK → products.id Sản phẩm phát sinh
movement_type string INDEX Loại biến động (IN/OUT/ADJUST...)
quantity decimal(18,3) NOT NULL Số lượng biến động
balance_after decimal(18,3) NOT NULL Tồn sau giao dịch
unit_cost decimal(18,2) NULL Đơn giá tham chiếu
transacted_at datetime INDEX phối hợp Thời điểm giao dịch
reference_code string NULL Mã tham chiếu chứng từ
note text NULL Ghi chú
created_by bigint FK → users.id, NULL Người tạo
created_at, updated_at timestamp Thời gian tạo/cập nhật
Index: (warehouse_id, product_id, transacted_at) với tên inventory_transactions_lookup_idx

4. Ràng buộc xóa (ON DELETE)

Quan hệ FK Hành vi xóa Giải thích
products.product_group_id → product_groups.id SET NULL Xóa nhóm không xóa sản phẩm, chỉ bỏ liên kết nhóm
inventories.(warehouse_id, product_id) CASCADE Xóa kho/sản phẩm sẽ xóa bản ghi tồn liên quan
stock_receipts.warehouse_id RESTRICT Không cho xóa kho nếu còn phiếu nhập
stock_receipts.created_by SET NULL Xóa user vẫn giữ phiếu, bỏ thông tin người tạo
stock_receipt_items.stock_receipt_id CASCADE Xóa phiếu nhập sẽ xóa dòng chi tiết
stock_receipt_items.product_id RESTRICT Không cho xóa sản phẩm nếu còn dòng phiếu nhập
stock_issues.warehouse_id RESTRICT Không cho xóa kho nếu còn phiếu xuất
stock_issues.created_by SET NULL Xóa user không làm mất phiếu xuất
stock_issue_items.stock_issue_id CASCADE Xóa phiếu xuất sẽ xóa chi tiết
stock_issue_items.product_id RESTRICT Không cho xóa sản phẩm nếu còn dòng xuất
stock_transfers.from_warehouse_id, to_warehouse_id RESTRICT Không cho xóa kho nếu có lịch sử chuyển kho
stock_transfers.created_by SET NULL Xóa user không làm mất phiếu chuyển
stock_transfer_items.stock_transfer_id CASCADE Xóa phiếu chuyển sẽ xóa chi tiết
stock_transfer_items.product_id RESTRICT Không cho xóa sản phẩm nếu còn dòng chuyển
stocktakes.warehouse_id RESTRICT Không cho xóa kho nếu còn phiếu kiểm kê
stocktakes.created_by SET NULL Xóa user không làm mất phiếu kiểm kê
stocktake_items.stocktake_id CASCADE Xóa phiếu kiểm kê sẽ xóa chi tiết
stocktake_items.product_id RESTRICT Không cho xóa sản phẩm nếu còn dòng kiểm kê
inventory_transactions.warehouse_id, product_id RESTRICT Giữ toàn vẹn lịch sử giao dịch
inventory_transactions.created_by SET NULL Xóa user không làm mất lịch sử
